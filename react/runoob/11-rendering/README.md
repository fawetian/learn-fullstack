# 11-rendering: React 渲染机制

来源: https://www.runoob.com/react/react-rendering.html

## 核心概念

1. **渲染流程** — React 应用启动时创建根节点（createRoot），根组件及其子组件递归渲染为虚拟 DOM，最终转换为真实 DOM 插入页面。
2. **触发重新渲染** — 状态变化（setState）、Props 变化、父组件重新渲染都会触发组件重新渲染。
3. **虚拟 DOM** — React 维护内存中的虚拟 DOM 树，每次渲染生成新树，通过 Diff 算法对比差异，只更新变化的部分（最小化 DOM 操作）。
4. **Diff 算法** — 对比两棵虚拟 DOM 树的差异时，React 采用同级比较策略，假设不同层级的节点很少跨级移动，提升比较效率。
5. **Keys 的作用** — 列表渲染时为每个元素提供稳定的唯一 key，帮助 React 识别元素身份，优化插入、删除和排序时的更新性能。

## 运行/学习方法

本章是原理说明章节，理解 React 的渲染机制对编写高性能组件至关重要。建议：

1. 阅读示例文件中的注释，理解 React 的渲染触发条件和更新流程
2. 在 `18-lists-keys` 章节中实践 key 属性的正确使用
3. 在 `26-memo` 章节中学习使用 `React.memo` 和 `useMemo` 优化不必要的重新渲染

关键记忆点：React 的"声明式"意味着你只需描述 UI 应该是什么样，React 会自动计算最小化的 DOM 更新步骤。
